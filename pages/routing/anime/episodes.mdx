# Fetching Episodes
Anify essentially fetches the episodes from every single provider via the . Technical details regarding the usage of getting episode lists from Anify can be found below. Please note that because Anify fetches episodes from every single provider, some response times may be slow for shows that contain a lot of episodes such as ONE PIECE if the response is not cached. The longest response time will be around 30 seconds for ONE PIECE specifically. Shows that only contain around 12-24 episodes are usually around 3 seconds long max.

## Route Schema (URL)
```
/episodes/{id}
```
| Parameter |  Type  | Description                                                       | Required? | Default |
| :-------: | :----: | ----------------------------------------------------------------- | :-------: | :-----: |
|   id   | string | The AniList ID for that media. |    Yes    |  `""`   |

```
/episodes?id={id}
```
| Parameter |  Type  | Description                                                       | Required? | Default |
| :-------: | :----: | ----------------------------------------------------------------- | :-------: | :-----: |
|   id   | string | The AniList ID for that media. |    Yes    |  `""`   |

### POST
```
/episodes
```
#### Body Example:
```json
{ "id": "{id}" }
```

Request can be either a `POST` or `GET` request.
```javascript
const axios = require("axios");

const url = "https://api.anify.tv/episodes/21";
const data = async () => {
    try {
        const { data } = await axios.get(url);
        return data;
    } catch (err) {
        throw new Error(err.message);
    }
};
 
console.log(data);
```
